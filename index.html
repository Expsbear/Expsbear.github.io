BFC清除浮动

BFC全称是块级格式上下文（Block Fromatting Context），它是按照块级盒子布局的。

BFC的主要特征：
1、BFC容器是一个隔离的容器，和其他元素互不干扰；所以我们可以用触发两个元素的BFC来解决垂直边距折叠问题。
2、BFC可以包含浮动；通常用来解决浮动父元素高度坍塌的问题。

BFC清除浮动就是利用了它“包含浮动”这一属性。当我们给浮动元素的父元素添加以下属性时，就能创建一个新的BFC，从而达到清除浮动的效果：

float 为 left、right
overflow 为 hidden、auto、scorll
display 为 table-cell、table-caption、inline-block、flex、inline-flex

1、脱离文档流。
2、向左/向右浮动直到遇到父元素或者别的浮动元素。

闭包

什么是闭包？闭包是指能够访问自由变量的函数。换句话说，在闭包中定义的函数可以”记忆”它被创建的环境。(注：自由变量是既不是在本地声明也不作为参数传递的一类变量，也可以这么说，如果一个作用域中使用的变量不是在该作用域中声明的，那么这个变量对于该作用域来说就是自由变量。)

执行上下文

执行上下文是一个抽象的概念，ECMAScript 规范使用它来追踪代码的执行。可以说这是代码第一次执行或执行的流程进入函数主体时所在的全局上下文。

一个执行上下文正在运行并不代表另一个执行上下文需要等待它完成运行之后才可以开始运行。有时会出现这样的情况，一个正在运行中的上下文暂停或中止，另外一个上下文开始执行。暂停的上下文可能在稍后某一时间点从它中止的位置继续执行。一个新的执行上下文被创建并推入栈顶，成为当前的执行上下文，这就是执行上下文替代的机制。

当我们有很多执行上下文一个接一个地运行时——通常情况下会在中间暂停然后再恢复运行，为了能很好地管理这些上下文的顺序和执行情况，我们需要用一些方法来对其状态进行追踪。而实际上也是如此，根据ECMAScript的规范，每个执行上下文都有用于跟踪代码执行进程的各种状态的组件，这里我们来讲一下词法环境。

词法环境

这里为什么要讲词法环境呢？因为词法环境明确声明它解决了执行上下文内代码中的“标识符引用”，这里的标识符可以想成是变量。

什么是词法环境？词法环境是一个基于 ECMAScript 代码的词法嵌套结构来定义特定变量和函数标识符的关联的规范类型。词法环境由一个环境记录及一个可能为空的对外部词法环境的引用构成。

其中，环境记录保留了所有存在于该词法环境中的标识符及其绑定的记录。每一个词法环境都有它自己的环境记录。而嵌套结构说明了一个内部环境引用了包围它的外部环境，同时，这个外部环境还可以有它自己的外部环境。结果就是，一个环境可以作为外部环境服务于多个内部环境。全局环境是唯一一个没有外部环境的词法环境。

总之，每个执行上下文都有一个词法环境。每次一个外围函数被调用时，就会创建一个新的词法环境。这个词法环境保留了变量和与其相关联的值，以及对其外部环境的引用。

那么现在我们知道，一个环境可以访问它的父环境，并且该父环境还可以继续访问它的父环境，以此类推，每个环境能够访问的一系列标识符，我们称其为“作用域”。我们可以将多个作用域嵌套到一个环境的分级链式结构中，即“作用域链”。

在上面的例子中，我们调用函数test，我们得到的值是 60，它也是调用函数 bar 的返回值（因为 foo 返回函数 bar）。即使 foo 已经返回了值，但是 bar 仍然可以访问自由变量 y，因为 bar 通过外部环境引用 y，这个外部环境即 foo 的环境！bar 还可以访问全局变量 x，因为 foo 的环境通向全局环境。这叫做“作用域链查找”。

总结：

2、每个环境能够访问的标识符集叫做“作用域”。我们可以将这些作用域嵌套成为一个分级的环境链——就是我们所知的“作用域链”。

3、每个函数都有一个执行上下文，它包括一个在函数中赋予变量含义的词法环境和对其父环境的引用。因为函数对环境的引用，使它看起来就像是函数“记住了”这个环境（作用域）一样，这就是一个闭包。

4、每当一个封闭的外部函数被调用时都会创建一个闭包。换句话说，内部函数不需要为了创建闭包而返回。

浅谈原型与原型链

2018-08-14

一、原型

在js世界里，存在着普通对象和函数对象，而这两种对象有什么区别呢？

在js所有函数对象中，都存在一个属性prototype，普通对象没有此属性，这个属性是每个构造函数创建的时候就会有的，而该属性指向它的原型对象（构造函数.prototype），而原型对象（构造函数.prototype）会默认生成一个constructor属性又指向构造函数。

在js所有对象中（包括普通对象和函数对象）都存在一个proto属性，该属性指向实例对象的构造函数的原型。

在js中，原型也是一个对象，原型的作用则是实现对象的继承。

结合上面的图和例子，应该可以更加通俗易懂了。在这个例子中，person,person1,person2都是构造函数Person创建的实例对象，设置了person1，person2的name属性，都可以读取到相应的值bear1，bear2，当删除它们的name属性时，从person1，person2中读取不到name属性时，就会从它们的原型中查找（也就是person1.proto == Person.prototype和person2.proto == Person.prototype），刚好找到了panda，构造函数创建的对象都会从原型”继承”属性，万一找不到呢？原型的原型又是什么呢？

在前面也提到了原型也是一个对象，而原型对象是通过Object构造函数生成的，原型对象的proto（即Person.prototype.proto）指向构造函数的prototype（即Object.prototype）,关系有点复杂了，我们还是简单粗暴直接上图：

到这里可能会有点疑问，那Object.prototype的原型呢？没错，就是null。

二、原型链

前面已经讲了有关原型的内容，那么对于原型链就非常好理解了。在查找一个对象的属性时，倘若在当前对象找不到该属性，则会沿着原型链一直往上查找，直到找到为止，如果到了原型链顶端，还没找到，则返回undefined。下图蓝色的这条线就形成了原型链：

补充:

当获取person.constructor时，其实person中并没有constructor属性,当不能读取到constructor属性时，会从person的原型也就是Person.prototype中读取，正好原型中有该属性，所以person.constructor == Person.prototype.constructor

总结：

1、所有的对象都有proto属性，该属性对应该对象的原型。

2、所有的函数对象都有prototype属性，该属性的值会被赋值给该函数创建的实例对象的proto属性。

3、所有的原型对象都有constructor属性，该属性对应创建所有指向该原型的实例的构造函数。

4、函数对象和原型对象通过prototype和constructor属性进行相互关联。

5、原型对象内创建的所有属性方法会被所有实例共享。

如何提交代码到码云和github上

2018-07-26

提交到码云上

  1. 安装Git，在本地创建一个项目文件夹，在该文件夹中右击打开Git Bash Here选项，在命令行窗口输入git init初始化库管理窗口。

  2. 修改Git的全局配置（下次不需修改）

git config –global user.name “你在码云的用户名”
git config –global user.email “你在码云的邮箱”

  1. 注册码云账号，在该平台上创建一个项目

  1. 将码云上的项目克隆到本地上，在命令行输入：

git clone 项目的https地址

  1. 创建远程库的配置信息,在git bash here命令行窗口下输入：

git remote add 远程库名（如origin） 项目的https地址
git remote -v //检查远程库名是否被使用
git remote rm //删除远程库名

  1. 在本地文件夹下创建文件（如index.js）

git add . //将文件添加到提交列表
git commit -m “注释” //将文件提交到本地仓库
git status //查看状态
git push -f 远程库名（如origin）master

注意：第一次推送代码到码云上时记得在push后面加上-f，之后推送可以不加-f

  1. 产生冲突时（即本地和远程同时修改同一区域代码），则会推送失败，此时可以输入以下命令，重新确定修改代码再进行推送：

git pull 远程库名（如origin） master

提交到github上

  1. 安装Git，在本地创建一个项目文件夹，在该文件夹中右击打开Git Bash Here选项，在命令行窗口输入git init初始化库管理窗口。

  2. 修改Git的全局配置（下次不需修改）

git config –global user.name “你在github上的账号”
git config –global user.email “你注册github的邮箱”

  1. 设置SSH key

首先检查是否已生成密钥cd ~/.ssh，如果返回的ls有3个文件,则密钥已经生成。

如果没有密钥，则可以通过以下命令生成：

ssh-keygen -t rsa -C “你注册github的邮箱”

生成过程中一路按3次回车键就好了。（默认路径，默认没有密码登录）
生成成功后，去对应目录C:\Users\hyt.ssh里（hyt为电脑用户名，每个人不同）用记事本打开id_rsa.pub，得到ssh key公钥。

  1. 为github账号配置ssh key

切换到github，展开个人头像的小三角，点击settings，然后打开SSH and GPG keys菜单， 点击New SSH key新增密钥，填上标题（最好跟本地仓库保持一致）

接着将id_rsa.pub文件粘贴到Key，最后Add SSH key生成密钥。

  1. 创建项目并关联github仓库

展开加号旁边的小三角，点击New repository,创建新项目。

接着执行以下命令：

git remote add 远程库名（如origin） 项目的https地址
git remote -v //检查远程库名是否被使用
git remote rm //删除远程库名

  1. 在本地文件夹下创建文件（如index.js）

git add . //将文件添加到提交列表
git commit -m “注释” //将文件提交到本地仓库
git status //查看状态
git push -u 远程库名（如origin）master

注意：第一次推送代码到码云上时记得在push后面加上-u，之后推送可以不加-f

  1. 产生冲突时（即本地和远程同时修改同一区域代码），则会推送失败，此时可以输入以下命令，重新确定修改代码再进行推送：

git pull 远程库名（如origin） master

其实，上传代码到码云和github的步骤基本一样，大同小异，只要注意一些小细节就可以了，比如说，上传到github上的话就需要添加密钥。好了，这篇文章就到此结束了，希望对你有所帮助！

如何区分css动画之animation、transition、transform、translate

2018-07-06

阿里有这样一道面试题：使用css实现一个持续的动画效果

我相信这道面试题对于大多数人来说都没什么问题的

这道题主要考察的是animation的用法，看到animation大家都应该会想到transition、transform、translate，那么问题来了，这四个css动画到底该怎么使用？有什么区别呢？

属性 含义
animation（动画） 用于设置动画属性，他是一个简写的属性，包含8个属性
transition（过渡） 用于设置元素的样式过度，和animation有着类似的效果，但细节上有很大的不同
transform（变形） 用于元素进行旋转、缩放、移动或倾斜，和设置样式的动画并没有什么关系，就相当于color一样用来设置元素的“外表”
translate（移动） translate只是transform的一个属性值，即移动。

如果弄清楚了他们的属性和含义，那么接下来我们就可以愉快的玩转css动画了。

transition

首先我们来看一个简单的demo：

我们来分析这一整个过程，首先transition给元素设置的过渡属性是transform，当鼠标移入元素时，元素的transform发生变化，那么这个时候就触发了transition，产生了动画，当鼠标移出时，transform又发生变化，这个时候还是会触发transition，产生动画，所以transition产生动画的条件是transition设置的property发生变化，这种动画的特点是需要“一个驱动力去触发”，有着以下几个不足：

  1. 需要事件触发，所以没法在网页加载时自动发生
  2. 是一次性的，不能重复发生，除非一再触发
  3. 只能定义开始状态和结束状态，不能定义中间状态，也就是说只有两个状态

语法：transition: property duration timing-function delay;

值 描述
transition-property 规定设置过渡效果的 CSS 属性的名称
transition-duration 规定完成过渡效果需要多少秒或毫秒
transition-timing-function 规定速度效果的速度曲线
transition-delay 定义过渡效果何时开始

anmation

在官方的介绍中这个属性是transition属性的扩展，可以把它理解为animation是由多个transition的效果叠加，从而做出复杂酷炫的效果，同样我们还是先来一个demo：

我们先来看看keyframes这个关键点，它定义了一个动画组合叫changebox，里面的10%，50%，70%，100%代表在变化中不同时间点的属性值，比如这个动画的总时间是1s，那么10%就是在0-0.1s中的动画，通过这个我们可以较精确的控制动画变化中任何一个时间点的属性效果，这大大提高了我们对动画的把控，是做复杂动画的基础，animation中有八个值，通过控制animation的每个值，控制动画变得非常灵活，我们来具体了解它的语法以及各个值代表着什么：

语法：animation: name duration timing-function delay iteration-count direction play-state fill-mode;

值 描述
name 用来调用@keyframes定义好的动画，与@keyframes定义的动画名称一致
duration 指定元素播放动画所持续的时间
timing-function 规定速度效果的速度曲线，是针对每一个小动画所在时间范围的变换速率
delay 定义在浏览器开始执行动画之前等待的时间，值整个animation执行之前等待的时间
iteration-count 定义动画的播放次数，可选具体次数或者无限(infinite)
direction 设置动画播放方向：normal(按时间轴顺序),reverse(时间轴反方向运行),alternate(轮流，即来回往复进行),alternate-reverse(动画先反运行再正方向运行，并持续交替运行)
play-state 控制元素动画的播放状态，通过此来控制动画的暂停和继续，两个值：running(继续)，paused(暂停)
fill-mode 控制动画结束后，元素的样式，有四个值：none(回到动画没开始时的状态)，forwards(动画结束后动画停留在结束状态)，backwords(动画回到第一帧的状态)，both(根据animation-direction轮流应用forwards和backwards规则)，注意与iteration-count不要冲突(动画执行无限次)

更多详细内容请参考：http://www.runoob.com/cssref/css3-pr-animation.html

transform与translate

为什么把transform和translate放在一起讲呢？上面也提到了translate是transform的一个属性值，所以放在一起讲再合适不过了，老规矩还是先来个例子：

首先这个例子transform使用的是translate属性，讲图形沿着x轴和y轴移动一定的距离，其实transform与translate用起来是非常简单的，所以这里就不多说了，现在介绍transform一些其他常用属性值：

语法：transform: none|transform-functions;

值 描述
none 定义不进行转换
translate（x，y） 定义沿着x轴和y轴进行移动
tanslateX（x） 定义沿着x轴进行移动
tanslateY（y） 定义沿着y轴进行移动
scale（x，y） 通过设置x和y的值来进行伸缩转换
scaleX（x） 通过设置x的值来进行伸缩转换
scaleY（y） 通过设置y的值来进行伸缩转换
rotate（angle） 通过设置angle的值来旋转一定的角度

更多的属性值可以参考： W3School

用hexo+github快速搭建个人博客

2018-07-03

安装node.js

这里以windows环境为例，到node.js官方，选择合适的版本进行下载并安装。

安装git客户端

到git官网选择合适的版本进行下载，注意勾选Git Bash Here和Git GUI Here选项，方便随时随地打开命令窗口。

github创建个人仓库

登录到GitHub,如果没有GitHub帐号，使用你的邮箱注册GitHub帐号,点击GitHub中的New repository创建新仓库，仓库名应该为：用户名.github.io 这个用户名使用你的GitHub帐号名称代替，这是固定写法，比如我的仓库名为：Expsbear.github.io

本地git与github账号绑定

右击鼠标点击Git Bash Here，输入以下命令设置配置信息：

git config –global user.name “你的GitHub用户名”
git config –global user.email “你的GitHub注册邮箱”

生成ssh密钥文件：

ssh-keygen -t rsa -C “你的GitHub注册邮箱”

然后直接三个回车即可，默认不需要设置密码

然后找到生成的.ssh的文件夹中的id_rsa.pub密钥，将内容全部复制

打开GitHub_Settings_keys 页面，新建new SSH Key

Title为标题，任意填即可，将刚刚复制的id_rsa.pub内容粘贴到Key下面的文本框，最后点击Add SSH key。

在Git Bash中输入以下命令检测gtihub公钥设置是否成功：

ssh git@github.com

设置GitHub密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当你推送的时候，git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

安装hexo

在电脑上创建一个文件夹blog，创建后进入文件夹中，按住shift键，右击鼠标点击“在此处打开Powershell窗口”，使用npm命令安装hexo，输入：

npm install -g hexo-cli

初始化hexo

接着输入以下命令初始化hexo：

hexo init

安装依赖包：

npm install

本地运行

npm g

npm s

在浏览器中输入localhost:4000，就可以看到博客搭建成功了

hexo的一些基本命令

hexo n “我的博客” == hexo new “我的博客” #新建文章

hexo g == hexo generate #用于生成静态文件

hexo s == hexo server #用于启动服务器，主要用来本地预览

hexo d == hexo deploy #用于将本地文件发布到github等git仓库上

hexo server #Hexo会监视文件变动并自动更新，无须重启服务器

hexo server -s #静态模式

hexo server -p 5000 #更改端口

hexo server -i 192.168.1.1 #自定义 IP

hexo clean #清除缓存，若是网页正常情况下可以忽略这条命令

hexo与github关联起来

打开blog文件夹下的配置文件_config.yml，翻到最后修改为：

deploy:
type: git
repo: 这里填入你之前在GitHub上创建仓库的完整路径，记得加上 .git
branch: master
参考如下：

deploy:
type: git
repo: https://github.com/Expsbear/Expsbear.github.io.git
branch: master

安装Git部署插件：

npm install hexo-deployer-git –save

这时，我们分别输入三条命令：

hexo clean
hexo g
hexo d

打开浏览器，在地址栏输入你的放置个人网站的仓库路径，即xxx.github.io，你就会发现你的博客已经上线了，可以在网络上被访问了。

更换主题

本次以Laughing为例，首先，由于Laughing主题使用pug模板引擎开发，所以首先需要在主程序中安装依赖组件：

npm install hexo-renderer-pug –save

拷贝主题到themes目录：

cd themes
git clone git@github.com:BoizZ/hexo-theme-laughing.git

修改主程序的_config.yml文件：

theme: hexo-theme-laughing

主题的配置都放在了_config.yml文件里面，在这里我就不展开来讲了，如果不喜欢本主题，可以到Themes选择自己喜欢的进行更换。
