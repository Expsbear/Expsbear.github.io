JavaScript中的this原理
更新于 2018-09-07

什么是this?

首先，我们来看一段代码：

在上面的代码中，obj.foo()和foo()虽然都指向了同一个函数，但结果却不一样，原因就在于this关键字，this指的是函数运行时所在的环境。对于obj.foo来说，foo运行在obj环境，因为foo左边是obj对象调用的，所以this指向obj；而对于foo()来说，foo运行在全局环境，因为foo指向了函数本身，没有对象调用了foo，所以this指向全局。总结出来下面一句话：

this 的值并不是由函数定义放在哪个对象里面决定，而是函数执行时由谁来唤起决定。

this与箭头函数

箭头函数的this和普通函数的this到底有什么区别呢？箭头函数按词法作用域来绑定它的上下文，所以this实际上会引用到原来的上下文，也可以这么说，箭头函数从包含它的词法作用域中继承到了this的值。说了这么多，我们还不如来看个例子吧：

到这里问题就来了，为什么double()里面的map函数里面的this指向window，而doubleArrow()里面的map函数里面的this却指向了object呢？首先我们来看double函数，传给map的匿名函数是谁调用的呢？这里没有对象调用它，因此this指向全局的window对象，而箭头函数中的this指向object是因为箭头函数按词法作用域将它的上下文绑定到原来的上下文。那么原来的上下文是什么呢？object.doubleArrow()中object调用了doubleArrow函数，因此原来的上下文就是object。

总结：

1、this的值并不是由函数定义放在哪个对象里面决定，而是函数执行时由谁来唤起决定。

2、箭头函数按词法作用域将它的上下文绑定到原来的上下文。
